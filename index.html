Deploy the Enterprise J2EE version (Docker / Render Guide)
Option A: Deploy to Render (Zero-Config SQLite fallback)
  • 1
    Create a free account on Render.com.
  • 2
    Create a new Web Service, and link your cloned GitHub repository amritanshu2711/AI-Object_Detection.
  • 3
    Set the runtime environment of the Web Service to Docker. Render will read our multi-stage Dockerfile.
  • 4
    Leave environment settings empty. Since no MySQL variables are present, the app will instantly boot and run on a self-contained SQLite database inside the container.
  • 5
    Render will start your application. Once online, visit the provided Render URL to access the full user signups, detection history, and admin reporting logs!
Option B: Run locally using Docker Compose (MySQL + Tomcat)
  • 1
    Open a terminal inside your cloned local repository directory.
  • 2
    Start the combined stack:
    docker-compose up --build
  • 3
    Docker will launch a MySQL database container (pre-seeded with schema.sql) and compile/run the J2EE application inside Tomcat.
  • 4
    Open your browser and load the site:
    http://localhost:8080/ai_object_detection_system
  • 5
    Log in as admin: Username: admin, Password: admin123.
